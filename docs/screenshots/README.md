# Screenshots

**What goes here:** concrete UI reference — actual interfaces and layouts.

Drop images that answer the question: *"what should this look like and how should it work?"*

Examples:
- Screenshots of apps with UI patterns you want to copy
- Competitor interfaces (to match or deliberately differ from)
- Specific components you like: navbars, cards, modals, tables, forms
- Layout structures: sidebar, hero, dashboard grid, list view
- Dark/light mode examples you want to reference

Screenshots set the **structural and component-level direction** — what Claude uses when designing layouts, picking component patterns, and making spacing decisions.

Use any format: `.png`, `.jpg`, `.webp`

---

> Run `/generate-visual-direction` after adding images. Claude synthesises both `moodboard/` and `screenshots/` into `docs/visual-direction.md` — a compact design brief it can reference every session without re-reading your images.
